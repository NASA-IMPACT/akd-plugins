# Contexts

Domain knowledge and reference vocabularies the planner grounds its decisions in.

- [`astro-domain-best-practices.md`](astro-domain-best-practices.md) — how astronomical data
  works: the four data types, the metadata that matters (WCS, exposure, calibration), FITS, the
  VO/ADQL query-then-download mindset, and data-quality caveats. Use this to classify a request
  and know what to ask for.
- [`operational-policies.md`](operational-policies.md) — the planner's fixed operating policies:
  input normalization, ADS/literature policy, name resolution, VO routing, archives in scope and
  priorities, fitness-to-use ordering, GCN ingestion, and output grouping.
- [`controlled-vocabularies.md`](controlled-vocabularies.md) — reference vocabularies (UAT, NASA
  SCaN acronyms) and the mission/instrument/platform lists used for routing and normalization.
