# Scope

## Purpose
An interview agent that systematically gathers system requirements by eliciting functional and scientific requirements, identifying stakeholder needs and success criteria, clarifying project goals and constraints, and defining the system scope before any design or implementation begins.

## Intended audience / visibility
- Output is intended for management to review.
- Information is provided (filled in) by developers and scientists who will start development work.

## Primary users (roles)
- Developers
- Scientists
- Management

## User expertise level
- Typically Master’s level and above

## Tasks supported
- Conduct structured interviews to extract requirements and rationale for the proposed development
- Elicit: why development is needed, current pain points, and how the system will solve the problem
- Identify stakeholders
- Ask/simulate stakeholder interaction questions (how stakeholders will interact with the system)
- Clarify end goal; define what success looks like and what failure looks like
- Define project scope: what is in-scope vs out-of-scope
- Clarify desired end state / target outcomes
- Produce a management-facing overview explaining why development is happening and a time/effort expectation (high-level)

## Current workflow
- No current workflow exists today (this step is not currently done).

## Pain points / bottlenecks
- No established workflow for developers to systematically think about end users / stakeholders before building
- Developers often lack direct access to users/stakeholders
- Practical constraints can prevent running a full discovery workshop at the start of every project or feature request

## Human-controlled decisions
- Final interview responses/requirements content must be provided/approved by humans
- The agent may provide examples, but humans decide what is included in the final answers

## Success criteria
- Produces a ~3–4 page scoping document at the end
- Document clearly captures project/software development requirements
- Document clearly states success criteria and the desired end state
