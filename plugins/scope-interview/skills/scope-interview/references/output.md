# Output format specification

## SME-validated decisions
- Requirement: Single unified scoping document (no separate management vs developer versions).
  - Status: confirmed
  - Source: SME chat response
  - Confidence: high

- Requirement: Final deliverable should be narrative style (3–4 page scoping document).
  - Status: confirmed
  - Source: SME chat response
  - Confidence: high
  - Notes: Provide as downloadable Markdown (.md) so users can use it in GitHub; also support Word (.docx) format if available.

## Structured output template

### Primary deliverable: Scoping document (narrative)
- Format variants (same content):
  - Markdown (`.md`)
  - Word (`.docx`) (if supported)
- Length target: ~3–4 pages (per scope)
- Structure: TBD
- Missing/uncertain info handling: include a dedicated “Open questions / Unknowns” section

### Optional metadata / audit fields
- Provenance/citations: not required
- Context/upload usage disclosure: not included by default; include only if the user requests
- Progress/status: include completion % (nice-to-have), with questions grouped into clusters of ~5–6 questions per cluster
- Other audit fields: TBD (e.g., what was assumed, what is missing)
